### Task 3: Fix chat template formatting in Colab notebook

**Files:**
- Modify: `notebooks/training/d4bl_fine_tuning.py:220-235` (the `format_and_tokenize` function)

This is the most impactful fix. The current `format_and_tokenize` manually concatenates `<|im_start|>role\ncontent<|im_end|>` strings, which loses the tokenizer's native chat template structure. Qwen2.5-Instruct has a built-in chat template that the model was pre-trained with — using it means the model sees the same token patterns during fine-tuning as it did during pre-training.

- [ ] **Step 1: Replace format_and_tokenize with native template version**

Replace the `format_and_tokenize` function (lines 220-235) with:

```python
def format_and_tokenize(dataset, tokenizer):
    """Convert messages-format dataset to text using the model's native chat template.

    Uses tokenizer.apply_chat_template() to produce the exact token format
    the model was pre-trained with, preserving instruction-following structure.
    Falls back to manual ChatML if the tokenizer lacks a chat template.
    """
    formatted = []
    for i in range(len(dataset)):
        msgs = dataset[i]["messages"]
        try:
            text = tokenizer.apply_chat_template(
                msgs, tokenize=False, add_generation_prompt=False
            )
        except Exception:
            # Fallback: manual ChatML (should not happen with Qwen2.5)
            parts = []
            for msg in msgs:
                parts.append(f"<|im_start|>{msg['role']}\n{msg['content']}<|im_end|>")
            text = "\n".join(parts)
        formatted.append({"text": text})
    return Dataset.from_list(formatted)
```
